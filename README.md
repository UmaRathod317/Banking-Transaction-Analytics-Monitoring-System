# Banking Transaction Analytics & Monitoring System

## Project Overview

This project analyzes banking debit and credit transactions to generate actionable insights into customer transaction behavior, branch performance, banking operations, transaction channels, and suspicious transaction patterns.

The project was developed using Microsoft Excel and MySQL, following a complete analytics workflow including data cleaning, CSV processing, database development, KPI reporting, transaction monitoring, and business intelligence analysis.

---

## Business Problem

Banks process thousands of debit and credit transactions daily across multiple branches and channels. Monitoring transaction volumes, customer activity, banking performance, and suspicious transactions is essential for operational efficiency and risk management.

This project aims to answer key business questions such as:

* What is the total credit and debit transaction volume?
* What is the net transaction amount across the banking network?
* Which branches generate the highest transaction volume?
* Which banks contribute the highest transaction value?
* Which transaction channels are most frequently used?
* How active are customer accounts?
* Are there suspicious transaction patterns requiring further investigation?

## SQL Views Developed

### Transactions_Per_DayWeekMonth

Analyzes transaction volume trends across daily, weekly, and monthly periods.

### Transaction_Amount_By_Branch

Measures branch-wise transaction performance.

### Transaction_Volume_By_Bank

Evaluates banking institution transaction contribution.

### Transaction_Method_Distribution

Analyzes customer transaction channel preferences.

### Branch_Transaction_Growth

Tracks branch-level transaction growth trends.
